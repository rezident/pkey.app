# PKey.App 🗝️

[![Website](https://img.shields.io/badge/website-pkey.app-blue)](https://pkey.app)

[pkey.app](https://pkey.app) — это автономный менеджер паролей в формате PWA. Работает без серверов, следует принципу нулевого разглашения (zero-knowledge) и отдает приоритет локальному хранению (local-first).

## 💡 Философия: никаких бэкендов и слежки

Мы убеждены: ваши самые конфиденциальные данные не должны лежать на чьих-то закрытых серверах. Поэтому у PKey радикальный подход — **здесь вообще нет бэкенда**.

PKey — это на 100% клиентское приложение. Под капотом только статические файлы, которые работают прямо в вашем браузере. Всё шифрование, расшифровка и управление данными происходят исключительно на вашем устройстве. Ничто не уходит в сеть без вашего ведома.

## ✨ Ключевые фичи (в планах и в разработке)

- 🔒 **Честный Zero-Knowledge**: Ваше хранилище лежит локально в `IndexedDB` устройства. Оно зашифровано ключом, который генерируется на основе вашего мастер-пароля.
- 📱 **Mobile-First и полный офлайн**: Приложение сделано как PWA. Просто добавьте его на домашний экран, и оно будет отлично работать без интернета. На десктопах интерфейс адаптируется, чтобы с удобством использовать широкие экраны.
- ☁️ **Своё облако**: Синхронизируйте зашифрованное хранилище между устройствами по WebDAV (для любителей self-hosting) или через официальные API облачных дисков (Google Drive, Яндекс.Диск).
- 🔀 **Умное слияние (Smart Merge)**: При конфликтах синхронизации PKey разруливает их бесшовно — аккуратно объединяет данные, сравнивая время изменения каждой отдельной записи.
- 🗂️ **Минималистичная структура**: Единая база данных. Вместо замороченной иерархии папок — плоская система разделов (тегов). Одна запись может лежать в нескольких разделах сразу или не принадлежать ни к одному.
- 🕒 **Быстрый доступ**: Оставьте вкладку открытой и разблокируйте хранилище за секунду с помощью PIN-кода.
- 🔑 **Встроенный TOTP**: Нативная поддержка генерации одноразовых паролей (кодов 2FA) прямо в приложении.

## 🛡️ Архитектура безопасности

Чтобы сделать полностью фронтендовое приложение по-настоящему безопасным, мы используем современные веб-стандарты:
- **Генерация ключа**: Для создания надежного мастер-ключа из пароля используется **Argon2id** (работает через WebAssembly). Это делает атаки полным перебором (brute-force) практически невыполнимыми.
- **Шифрование**: Используем нативный **WebCrypto API (AES-GCM)** для молниеносного криптостойкого шифрования ваших данных.
- **Кастомный формат**: Внутри хранилище представляет собой проприетарный зашифрованный JSON. Это дает максимальную гибкость и при этом сохраняет минимальный вес файла для быстрой синхронизации.

## 🛠️ Стек технологий

- **Фреймворк**: React + TypeScript
- **Стейт-менеджер**: Zustand
- **Сборщик**: Vite
- **Криптография**: WebCrypto API + Argon2 (WASM)

## 🤍 Монетизация

PKey — это **Open Source** проект. Любой желающий может склонировать репозиторий, запустить у себя или захостить на своем домене (ведь это просто статика).

Чтобы поддерживать разработку, PKey использует модель **Honorware**. В официальной сборке на `pkey.app` некоторые «Pro»-функции могут быть закрыты добровольным донатом. Но так как код открыт, вы абсолютно свободны форкнуть репозиторий и разблокировать для себя всё. Спонсорство здесь — это просто способ сказать автору *«Спасибо»*, а не жесткий пэйвол.

## 🚀 Быстрый старт

```shell
# Клонируем репозиторий
git clone https://github.com/rezident/pkey.app.git

# Переходим в папку проекта
cd pkey.app

# Устанавливаем зависимости
npm install

# Запускаем локальный сервер
npm run dev
```
